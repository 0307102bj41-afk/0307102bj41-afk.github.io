01조명·색상 과의존으로 인한 오분류
문제 상황

어두운 조명에서 촬영된 깨끗한 방을 Dirty로 오분류하는 사례가 반복됐습니다. 모델이 물체의 배치·윤곽이 아니라 이미지의 색감과 밝기에 과도하게 의존했고, 같은 방을 밝게 찍으면 Clean, 어둡게 찍으면 Dirty로 판정이 뒤집히는 구조적 문제였습니다.

해결 과정

입력 전처리에 RGB → Grayscale → 3채널 복원 파이프라인을 추가해, 모델이 색상·조명을 사용하지 못하고 물체의 형태와 공간 배치에만 집중하도록 강제했습니다. RandomFlip·RandomRotation·RandomZoom 증강을 병행해 다양한 조명·각도에 대한 일반화도 함께 끌어올렸습니다.

기대 효과

조명 차이로 판정이 뒤집히는 현상이 해소되어 촬영 환경과 무관하게 일관된 청결도 판정이 가능해졌습니다. "어두우니 더럽다"가 아닌 "물체가 흩어져 있으니 더럽다"를 학습하도록 교정한 것이 성능 확보의 토대가 됐습니다.

02극단 확신값에서 드러난 과적합
문제 상황

학습이 진행될수록 Dirty 99% / Clean 98% 같이 0과 1에 치우친 극단 확률이 다수 출력됐습니다. 검증 데이터에서도 비슷한 자신감을 보였지만, 이는 모델이 일반 패턴이 아닌 훈련셋 자체를 외운 과적합의 전형적 신호였습니다.

해결 과정

EarlyStopping(monitor=val_loss)으로 검증 손실이 개선되지 않으면 학습을 조기 종료했습니다. 도메인 특성에 맞춰 방은 patience=3, 책상은 patience=5로 조정하고, Dropout(0.5/0.3)과 2단계 파인튜닝으로 과적합을 다중으로 억제했습니다.

방: Dense(256)·Dropout(0.5)·patience=3 책상: Dense(128)·Dropout(0.3)·patience=5
기대 효과

훈련 데이터 암기가 억제되어 처음 보는 이미지에 대한 일반화 성능이 향상됐습니다. 도메인별 patience 분리로 책상의 다양한 구도까지 학습하면서도 과적합에 빠지지 않는 균형을 확보했습니다.

03크롤링 데이터 편향과 잘못된 주의 영역
문제 상황

사람이 보기엔 명백히 깨끗한 이미지를 모델은 다르게 판단하는(Dirty 99% / Clean 92%) 인식 차이가 나타났습니다. Google 이미지 크롤링 데이터의 구도·스타일 편향이 원인으로, 모델이 청결도와 무관한 영역을 보고 있을 가능성이 컸습니다.

해결 과정

Grad-CAM 히트맵으로 모델의 시선을 직접 들여다본 결과, 배경·벽·창문 등 무관한 영역에 과도하게 주목하는 이미지 유형을 특정했습니다. 이 유형을 학습셋에서 제거하거나 해당 유형 데이터를 추가 수집해 재학습함으로써, 주의 집중을 실제 어질러진 물체 쪽으로 교정했습니다.

기대 효과

Grad-CAM을 단순 시각화가 아니라 데이터 품질 개선 도구로 활용한 사례입니다. 모델이 무엇을 보고 판단하는지를 근거로 데이터를 보강하는 루프를 만들어, 사람의 직관과 모델 판단 사이의 간극을 좁혔습니다.

04TensorFlow 버전 차이로 인한 추론 불일치
문제 상황

동일한 모델·동일한 이미지인데 OS마다 추론 결과가 달랐습니다. Windows(TF 2.10)는 Dirty 98%, macOS(TF 2.15)는 Clean 63%로 결과가 뒤집힐 정도였습니다. 프레임워크 버전 차이가 추론 결과에까지 영향을 줄 줄은 예상하지 못한 부분이었습니다.

해결 과정

OS별 지원 버전이 달라 통일이 불가능했기에, TF 버전을 통일할 수 있는 Linux(Ubuntu) + CUDA GPU 환경으로 일원화하고 모든 모델을 재학습했습니다. 학습된 가중치(.h5)는 Flask 서버에 탑재해, 접속 OS와 무관하게 서버가 단일 결과를 내도록 추론을 서버 측으로 통합했습니다.

기대 효과

추론을 서버로 일원화하면서 OS·환경에 따른 결과 편차가 원천 제거됐습니다. 학습·배포 환경 표준화의 중요성을 체감했고, 클라이언트는 가벼운 요청만 보내고 모델은 서버에서 일관되게 동작하는 안정적인 구조를 확보했습니다.

01ADC 변환 미완료 상태에서 읽은 쓰레기 값
문제 상황

조이스틱 값이 간헐적으로 튀거나 비정상적인 숫자로 읽혔습니다. ADC 변환이 끝나기도 전에 데이터 레지스터를 읽으면서 직전 변환 잔여값이나 미완료 값이 그대로 들어왔고, 모터가 정지 명령에도 미세하게 움직이는 등 입력 전체의 신뢰도가 흔들렸습니다.

해결 과정

변환 시작 후 곧바로 읽지 않고 변환 완료 플래그(EOC)가 set 될 때까지 폴링으로 대기한 뒤 데이터 레지스터를 읽도록 시퀀스를 수정했습니다. 샘플링 시간과 클럭 분주도 입력 임피던스에 맞게 조정해 변환 자체의 안정성을 확보했습니다.

while(!(ADC1->SR & ADC_SR_EOC)); // 변환 완료 대기 value = ADC1->DR; // 안정화된 값 읽기
기대 효과

입력값이 안정화되어 조이스틱 → 모터 제어로 이어지는 전체 체인의 신뢰도가 확보됐습니다. 데이터시트의 타이밍 다이어그램을 직접 읽고 시퀀스를 맞추며, 라이브러리가 가려주던 하드웨어 동작 원리를 체득했습니다.

02I2C 초기화 타이밍 불일치로 인한 LCD 깨짐
문제 상황

전원을 켤 때마다 LCD에 깨진 문자가 출력되거나 아무것도 표시되지 않는 현상이 비결정적으로 발생했습니다. I2C 변환 모듈과 MCU 사이의 초기화 순서·타이밍이 LCD 컨트롤러가 요구하는 사양과 맞지 않은 것이 원인이었습니다.

해결 과정

LCD 컨트롤러 데이터시트가 요구하는 전원 안정화 지연과 초기화 명령 시퀀스(4bit 모드 전환·기능 설정 순서·각 단계 간 지연)를 정확히 맞춰 재구성했습니다. I2C ACK 확인을 추가해 각 명령이 실제 전달됐는지 검증한 뒤 다음 단계로 진행하도록 했습니다.

기대 효과

매 부팅마다 LCD가 안정적으로 초기화되어 상태 표시 UI를 믿고 쓸 수 있게 됐습니다. "동작은 하는데 가끔 깨진다"는 비결정적 버그를 타이밍 사양 준수로 해결하며 통신 모듈 디버깅의 표준 접근법을 익혔습니다.

03UART 수신 무한 대기 & 패킷 깨짐
문제 상황

블루투스 통신 중 신호가 잠깐 끊기거나 노이즈가 끼면 수신 루프가 오지 않는 바이트를 기다리며 멈춰버리는 현상이 있었습니다. 패킷 경계가 어긋나면 방향·버튼 데이터가 뒤섞여 엉뚱한 제어 명령이 실행되기도 했습니다.

해결 과정

타임아웃과 예외 핸들러를 도입해 일정 시간 내 데이터가 오지 않으면 대기를 빠져나와 안전 상태(정지)로 복귀하도록 했습니다. 더불어 상태 머신 기반 패킷 파서로 헤더 → 데이터 → 체크섬을 단계별 검증하고, 체크섬이 어긋나면 패킷을 폐기해 통신 신뢰성을 확보했습니다.

기대 효과

통신 끊김 상황에서도 시스템이 멈추지 않고 안전하게 복구되어 무선 환경의 불안정성에 견디는 펌웨어가 됐습니다. 체크섬·타임아웃·상태 플래그라는 통신 무결성 설계 패턴을 직접 구현하며 예외 처리 감각을 길렀습니다.

01동작 완료 시간이 초기화되지 않고 누적되는 문제
문제 상황

공정별 동작 완료 시간(택타임)을 측정하도록 회로를 구성했지만, 공정이 끝난 뒤에도 시간 값이 계속 누적되어 실제 동작 시간과 전혀 다른 값이 HMI에 표시됐습니다. 시간을 저장하는 레지스터가 공정 시작 시점에 초기화되지 않아 이전 공정의 측정값이 그대로 남는 구조가 원인이었습니다.

해결 과정

SM412(1초 클록)와 INCP 명령으로 레지스터를 1초 단위로 증가시키던 회로에, 공정 시작 조건을 추가해 동작 시작 시 측정 레지스터(D412)를 0으로 초기화(MOV K0)하도록 수정했습니다. 매 공정마다 시간 측정이 0에서 새로 시작되도록 한 것입니다.

동작 시작 → MOV K0 D412 (레지스터 클리어) SM412(1초) + INCP → D412 누적 공정 종료 → 측정 정지
기대 효과

각 공정의 실제 동작 시간만 정확히 측정되도록 개선되어 공정 시간 모니터링의 신뢰성을 확보했습니다. 신뢰할 수 있는 택타임 데이터는 공정 효율 분석과 작업 조건 개선의 근거 자료로 활용할 수 있게 됐습니다.

02선택 비움·전체 비움 동시 동작으로 인한 서보 위치 충돌
문제 상황

선택 비움과 전체 비움 기능은 동일한 서보 위치 이동 알고리즘을 공유하기 때문에, 두 기능이 동시에 트리거되면 서로 다른 목표 위치를 동시에 요구하며 서보가 오동작할 가능성이 있었습니다. 적재·배출 위치가 어긋나면 소재 관리 오류로 직결되는 위험한 상황이었습니다.

해결 과정

선택 비움 전용 메모리를 B접점(NC) 방식의 인터락으로 구성했습니다. 선택 비움이 동작하면 해당 접점이 OFF가 되어 전체 비움 로직의 실행 라인을 차단하도록 설계해, 두 기능이 물리적으로 동시에 성립할 수 없게 만들었습니다. 선택 적재·배출 이후 자동 적재를 제한하는 모드 인터락도 함께 적용했습니다.

기대 효과

두 기능이 구조적으로 충돌할 수 없도록 원천 차단되어 동시 동작에 의한 위치 오류 가능성을 제거했습니다. 같은 알고리즘을 공유하면서도 인터락으로 동작을 명확히 구분하는 설계를 통해 시스템의 동작 안정성·신뢰성·유지보수성을 함께 끌어올렸습니다.

03층별 절대 위치 개별 저장의 비효율
문제 상황

초기 설계에서는 1·2·3층 각 위치를 모두 절대 좌표(ABS)로 개별 저장했습니다. 그 결과 설정해야 할 위치 데이터가 많아지고, 창고 높이나 기구 조정으로 위치가 바뀌면 여러 층의 좌표를 일일이 다시 수정해야 해 유지보수 부담이 컸습니다.

해결 과정

위치결정 데이터에 AXIS Pointer INC(상대 위치) 방식을 적용해, 각 층의 절대 좌표를 따로 저장하는 대신 층간 높이 값만 설정하도록 구조를 바꿨습니다. 기준점에서 층간 간격만큼 반복 이동하는 방식으로 전환하고, 티칭 기능을 연계해 현장에서 위치 값을 유연하게 보정할 수 있도록 했습니다.

기대 효과

관리할 설정값이 크게 줄고, 위치 변경 시 프로그램 수정 없이 층간 높이 한 값만 조정하면 되도록 개선됐습니다. 유지보수성과 확장성이 향상되었고, 절대/상대 위치 제어의 장단점을 직접 비교하며 위치 제어 설계 감각을 키웠습니다.
